CORE · PF02

접수·검증·저장·알림

문의나 신청을 안전하게 받고, 유실 없이 다음 담당자에게 넘기는 기본 자동화입니다.

  • 폼 또는 웹훅 입력 1종과 필수값 검증
  • 중복·인증 실패·알림 실패를 성공 건과 분리
  • Google Sheets 또는 시트형 결과, Slack 또는 Email 알림 구성

공개 체험 경계 실제 Google Sheets·Slack·Email에 전송하지 않습니다. 로컬 기기 알림은 사용자가 허용한 경우에만 비식별 결과를 표시합니다.

사례 상세체험 데모

OPTION · PF01

내용 판단이 필요할 때 AI 확장

접수된 문의를 읽고 요약, 유형, 우선순위, 담당팀, 답변 초안을 처리 목록으로 만듭니다.

  • 반복 문의의 1차 정리와 긴급 건 우선 확인
  • 판단 결과와 답변 초안을 담당자 검토용으로 제공
  • 합의한 AI API, 사내 API 또는 온프레미스 모델 연동 가능

공개 체험 경계 외부 AI가 없는 키워드 점수 방식입니다. 실제 AI 연동은 의뢰인이 제공하거나 사용을 승인한 계정으로 검증하며, 사용료·보안 정책·인간 검토 범위는 별도 협의합니다.

사례 상세체험 데모

OPTION · PF04

접수 후 처리가 남을 때 관리 화면 확장

단순 알림으로 끝나지 않고 승인·반려, 상태 필터, 처리 이력이 필요한 업무에 가벼운 웹 도구를 더합니다.

  • 신청 등록·필수값 검증과 목록·조건 필터
  • 승인·반려 상태 처리와 CSV 이력 저장
  • 업무 규칙에 맞춘 관리자 화면으로 확장

공개 체험 경계 브라우저 메모리의 가상 신청으로 동작합니다. 실제 계정·권한, 서버 DB, 인사·전자결재 시스템 연동은 별도 설계 범위입니다.
